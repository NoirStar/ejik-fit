# 이직핏 UI/UX·가독성·공개 신뢰성 감사

작성일: 2026-07-10

대상: `apps/web` 홈 대시보드, 기술 맵, 공고 상세, 공통 내비게이션

상태: 코드 및 실제 브라우저 렌더링 기준 감사 완료

## 1. 결론

현재 이직핏 웹은 첫인상과 기본 정보 구조는 준수하지만, 외부 공개 서비스로는
신뢰성과 상호작용 완결성이 부족하다. 화면은 잘 정리된 SaaS 대시보드처럼 보이지만,
실데이터와 샘플 데이터가 섞이고 근거 없는 정밀 수치가 표시되며 일부 조작이 동작하지
않는다. 사용자가 이를 한 번 발견하면 공식 채용 출처와 원문 근거라는 제품의 실제
강점까지 의심하게 된다.

현재 단계는 다음처럼 평가한다.

| 항목 | 점수 | 판단 |
| --- | ---: | --- |
| 시각 완성도 | 7/10 | 첫 화면과 다크 그래프의 인상은 좋음 |
| 가독성 | 7/10 | 정보 계층은 분명하지만 작은 보조 글자와 표 overflow가 있음 |
| 사용자 편의성 | 5/10 | 필터·스택 편집은 있으나 무동작 UI와 모바일 단절이 큼 |
| 브랜드 고유성 | 4/10 | 전형적인 SaaS 대시보드 문법과 페이지 간 불일치가 남음 |
| 공개 신뢰도 | 3/10 | 샘플·고정 수치와 가짜 계정 표현이 가장 큰 위험 |
| 외부 공개 준비도 | 4/10 | 내부 데모로는 가능하지만 공개 베타 전 재구축이 필요 |

## 2. 감사 방법

### 2.1 정적 코드 감사

- Next.js App Router 구조와 페이지별 metadata
- 홈 대시보드 데이터 조합과 상태 처리
- 내비게이션, 링크, 버튼, 필터, 스택 저장
- 기술 그래프 렌더러, force simulation, LOD, 대규모 fixture와 benchmark
- 전역 CSS, 반응형 분기, 접근성 속성
- 공개 정책, 문의, 개인정보, SEO 관련 파일

### 2.2 실제 브라우저 확인

- 홈: 1440×1000 라이트 모드
- 홈: 1440×1000 다크 모드
- 홈: 390×844 모바일 라이트 모드
- 기술 맵: 1440×1000 다크 모드
- 기술 맵: 390×844 모바일 다크 모드
- 본문 폭, 세로 길이, overflow, 콘솔 오류, 랜드마크 수 확인

검증 당시 홈 일부 API와 기술 맵 API는 정상 데이터를 제공하지 못했다. 이 상황에서
화면이 장애를 어떻게 전달하는지까지 감사 범위에 포함했다.

## 3. 현재 강점

### 3.1 명확한 홈 정보 순서

홈은 주간 요약, 시장 변화, 신규 공고, 관련 기술, 마감 공고 순으로 읽힌다. 사용자가
오늘 확인할 내용을 빠르게 훑는 기본 정보 구조는 좋다.

### 3.2 공식 출처와 원문 근거

공고 상세는 공식 원문 URL, 마지막 확인 시각, 필수·우대 기술, 원문 근거를 제공한다.
이는 이직핏이 일반 공고 모음 서비스와 구별될 수 있는 가장 중요한 신뢰 자산이다.

### 3.3 기술 맵의 시각적 잠재력

딥 네이비 캔버스, 노드 halo, 선택 주변 강조는 제품에서 가장 기억에 남는 시각
자산이다. 그래프가 화면의 주인공이 되도록 주변 UI와 렌더링 엔진을 재구성하면
브랜드를 대표하는 기능이 될 수 있다.

### 3.4 기본 접근성 기반

문서 언어, skip link, `prefers-reduced-motion`, 일부 focus 상태, 네이티브 select,
폼 레이블, 404와 loading 화면이 마련되어 있다. 완성도는 부족하지만 재구축 시
보존할 기반은 있다.

## 5. 높은 우선순위 UX 문제

### 5.1 동작하지 않는 조작

- 홈의 7일·30일·90일 버튼은 데이터 기간을 변경하지 않음
- 알림과 도움말 버튼은 동작하지 않음
- `전체 보기`는 현재 섹션으로 다시 이동
- 홈 공고 행은 상세 페이지로 이동하지 않음
- 기술 맵 도구 레일 버튼은 모두 동작하지 않음
- 상세 헤더의 `공고`, `트렌드`, `로드맵`은 존재하지 않는 anchor를 가리킴

관련 근거:

- `apps/web/src/components/skill-graph-experience.tsx:441-461`
- `apps/web/src/components/site-header.tsx:11-16`

### 5.2 가짜 계정 상태

계정 기능이 구현되지 않았지만 첫 방문자에게 고정 사용자명 `김민준`, 아바타,
알림 배지, 드롭다운이 표시된다. 기본 Java·Spring·AWS·Docker·Kubernetes 스택도
사용자가 선택하기 전에 개인 데이터처럼 나타난다.

관련 근거:

- `daily-dashboard-home.tsx:590-600`
- `apps/web/src/lib/owned-skills.ts:3-9`

익명 도구라는 사실과 로컬 저장 범위를 설명하고 첫 진입 스택 선택을 제공해야 한다.

### 5.3 모바일에서 핵심 기능 소실

820px 이하에서 사이드바 footer가 숨으면서 `내 스택 요약`과 스택 편집 기능도 함께
사라진다. 상단 내비게이션은 텍스트를 숨기고 `aria-hidden` 아이콘만 남겨 처음
사용하는 사람과 보조 기술 사용자 모두 의미를 파악하기 어렵다.

관련 근거:

- `apps/web/src/app/globals.css:6831-6873`

### 5.4 공고 표 overflow

1440px 브라우저에서도 긴 위치 문자열이 셀 폭을 넘어 제목과 날짜 영역을 침범했다.
모바일에서는 카드로 전환되어 가로 overflow는 없었지만 페이지 전체 길이가 약
2,841px까지 늘어났다.

### 5.5 기술 맵 모바일 작업 순서

390px 기술 맵은 그래프와 보조 카드가 먼저 나오고, 스택·필터·그래프 설정과 선택
인스펙터가 아래에 연속 배치된다. 사용자가 먼저 조작해야 할 필터가 결과 화면보다
뒤에 있어 탐색 순서가 반대다. 전체 문서 높이는 약 2,606px였다.

## 6. 기술 그래프 성능 감사

### 6.1 현재 렌더러의 한계

현재 렌더러는 `force-graph` 기반 Canvas 2D와 d3-force를 사용한다. 코드 자체가
현재 adapter를 대규모 그래프 미지원으로 표시한다.

- `apps/web/src/lib/graph-renderer.ts:55`: 대규모 기준 5,000노드
- `graph-renderer.ts:58-65`: `supportsLargeGraph: false`
- 5,000노드 이상에서 대체 WebGL adapter를 선택하는 계약만 있고 실제 adapter 없음

### 6.2 메인 스레드 지속 부하

애니메이션 모드에서 force simulation과 redraw가 무기한 실행된다. hover 시 전체
링크를 순회해 이웃을 찾고 simulation을 다시 가열한다. 노드와 링크가 증가하면
pan·zoom·hover가 모두 메인 스레드 force 계산과 경쟁한다.

관련 근거:

- `apps/web/src/components/skill-graph-force-canvas.tsx:255-292`
- `skill-graph-force-canvas.tsx:372-403`

### 6.3 연결되지 않은 LOD

zoom별 라벨과 간선을 제한하는 `graph-lod.ts`가 존재하지만 실제 화면 렌더러에서
호출되지 않는다. 현재 대규모 benchmark도 실제 WebGL/Canvas 렌더링, FPS, long
task, 메모리를 측정하지 않고 renderer 선택과 artifact 변환만 측정한다.

관련 근거:

- `apps/web/src/lib/graph-lod.ts`
- `apps/web/src/lib/large-graph.bench.ts`

따라서 현재 5,000노드 기준은 보장된 성능 예산이 아니라 대체 렌더러가 필요하다는
경고선이다.

## 7. 시각 시스템과 AI 제작 인상

일반 사용자가 첫눈에 AI 제작이라고 단정할 정도는 아니지만 다음 패턴이 누적되어
전문가에게는 프로토타입 인상을 준다.

- 사이드바, 파스텔 hero, KPI, 2열 카드, pill badge의 전형적인 SaaS 조합
- 회사 로고 대신 첫 글자와 순서 기반 색상을 사용하는 목업 표현
- 근거 없이 정밀한 퍼센트와 그래프
- 기능처럼 보이지만 동작하지 않는 컨트롤
- 홈, 기술 맵, 공고 상세의 서로 다른 색상·radius·내비게이션
- 7,026줄 단일 CSS에 여러 리디자인 override가 누적된 구조

AI 인상을 줄이려면 장식을 더 추가하는 것이 아니라 데이터의 근거와 상태를 정직하게
표시하고, 동작하는 기능만 노출하며, 하나의 시각 시스템을 사용해야 한다.

## 8. 타이포그래피와 접근성

- Geist는 Latin subset만 로드하고 한글은 운영체제 fallback에 맡김
- 720, 780, 820, 850 같은 비표준 굵기가 한글에서 합성될 수 있음
- 일부 11~12px 보조 문구와 placeholder 대비가 낮음
- 홈과 그래프에서 `<main>`이 중첩됨
- 그래프 캔버스가 `aria-hidden`이고 키보드 대체 탐색이 없음
- 모바일 내비게이션에서 텍스트가 사라짐
- 그래프 검색 입력의 접근 가능한 이름이 부족함

## 9. 공개 신뢰 기반 누락

- 개인정보·로컬 저장 안내 없음
- 이용 안내와 분석 지표 설명 없음
- 오류 신고와 기업 정정·삭제 요청 경로 없음
- 홈과 기술 맵에서는 footer가 숨겨짐
- OG, Twitter, canonical, sitemap, robots, manifest가 없음
- 공고별 metadata와 `JobPosting` JSON-LD가 없음

현재 footer는 공개 정보 수집 문구와 GitHub 링크만 제공한다.

## 10. 근본 원인

1. API 상태와 화면 표현 사이에 명시적인 상태 모델이 없음
2. 실데이터 부족을 UI 샘플로 감추는 설계
3. 홈과 그래프 기능이 각각 대형 client component에 집중됨
4. 공통 디자인 토큰과 앱 셸 없이 페이지별 override가 누적됨
5. 대규모 그래프 계약과 artifact는 있으나 실제 렌더러 통합이 완료되지 않음
6. 테스트가 존재 여부 중심이고 데이터 정직성·실제 동작·브라우저 성능을 검증하지 않음

## 11. 승인된 개선 방향

사용자와 다음 방향을 확정했다.

- 백엔드 수집기, API 계약, 스킬 추출, Fit 계산, 그래프 도메인 로직은 보존
- 프론트엔드 표현 계층은 새 앱 셸과 디자인 시스템으로 재구축
- 샘플·고정 수치·가짜 계정·무동작 UI 제거
- 홈, 공고, 기술 맵, 신뢰 페이지를 하나의 내비게이션과 스타일로 통합
- 기술 맵은 Sigma.js WebGL, Graphology, ForceAtlas2 Worker, 실제 LOD로 교체
- 데스크톱 50,000노드·150,000간선 엔진 용량을 목표로 함
- 공고 근거 노드는 선택 시 지연 노출하고 전체 그래프에는 모두 넣지 않음
- 로컬 Pretendard Variable과 Geist를 조합해 한글·숫자 품질을 통일
- 데이터 정책, 분석 지표, 개인정보·로컬 저장, 정정 요청, SEO를 공개 화면에 추가

상세 설계는
`docs/superpowers/specs/2026-07-10-frontend-rebuild-and-performance-graph-design.md`를
기준으로 한다.
