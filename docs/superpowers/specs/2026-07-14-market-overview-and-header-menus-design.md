# 이직핏 시장 화면과 헤더 메뉴 레이어 설계

## 상태

- 작성일: 2026-07-14
- 범위: 데스크톱 헤더의 알림·사용자 메뉴 배치 교정, `/market` 실제 데이터 화면 구현
- 선행 기준: `2026-07-13-community-feed-home-design.md`
- 승인: 사용자가 권장안으로 중단 없이 구현하도록 승인함

## 목표

1. 데스크톱에서 알림 및 사용자 메뉴가 2단 글로벌 내비게이션과 겹치지 않게 한다.
2. 준비 중 셸인 `/market`을 기존 API만 사용하는 실제 채용 시장 스냅샷으로 교체한다.
3. 역사 데이터가 없는 상태에서 추세나 변화율을 꾸며내지 않는다.
4. 부분 API 실패, 빈 데이터, 좁은 화면에서도 핵심 탐색 경로를 유지한다.

## 확인된 현재 상태

- 데스크톱 헤더는 64px 유틸리티 행과 48px 주요 탐색 행으로 구성된다.
- 알림 메뉴는 유틸리티 버튼을 기준으로 `top: calc(100% + 0.625rem)`에 배치되어, 실제 측정에서 주요 탐색 행과 47.5px 겹친다.
- 알림 메뉴와 사용자 메뉴는 같은 `.menu` 스타일을 사용하므로 한 규칙으로 함께 교정할 수 있다.
- 820px 이하에서는 데스크톱 탐색 행이 사라지고, 520px 이하에서는 메뉴가 이미 고정 위치로 전환된다.
- `/market`은 현재 `RouteShell` 기반 준비 중 화면이다.
- 기존 API는 `/api/postings`와 `/api/skills/stats`를 통해 공개 공고 수, 기술별 공고 수, 필수·우대·미분류 건수를 제공한다.
- 기존 API에는 기간별 스냅샷이나 이전 기간 비교 데이터가 없다.

## 검토한 접근

### 선택: 기존 메뉴 앵커 유지 + 실제 데이터 스냅샷 시장 화면

- 데스크톱에서만 메뉴의 세로 오프셋에 두 번째 탐색 행 높이와 여백을 반영한다.
- 기존 바깥 클릭, Escape, 포커스 복귀 로직과 수평 정렬은 보존한다.
- 시장 화면은 공고와 기술 통계를 병렬로 호출하고, 확인 가능한 현재 값만 보여준다.
- 변경 범위가 작고 기존 상호작용 및 API 계약을 보존한다.

### 보류: 메뉴를 포털 또는 전역 오버레이 루트로 이동

- 스크롤 컨테이너와 clipping에는 가장 강하지만 포커스·위치 계산·SSR 경계가 복잡해진다.
- 현재 문제는 데스크톱 세로 기준점 하나로 재현되므로 이번 범위에는 과하다.

### 보류: 역사 데이터 API부터 구축

- 실제 증감률과 시계열 차트를 제공할 수 있지만 수집 스냅샷, 저장 모델, 집계 계약이 먼저 필요하다.
- 현재 화면 구현과 독립된 백엔드 프로젝트로 분리한다.

## 헤더 메뉴 동작

### 데스크톱 821px 이상

- 알림 및 사용자 메뉴의 상단은 전체 헤더 하단보다 최소 8px 아래에 위치한다.
- 메뉴의 오른쪽 정렬은 각 버튼 앵커를 그대로 따른다.
- 메뉴가 열린 동안 주요 탐색 링크는 메뉴와 시각적으로 겹치지 않는다.
- 알림과 사용자 메뉴는 동시에 열리지 않는다.
- Escape, 바깥 클릭, 경로 이동 시 닫히고 포커스가 원래 버튼으로 돌아간다.

### 태블릿 521–820px

- 주요 탐색 행이 없으므로 현재 버튼 앵커 기준 배치를 유지한다.
- 검색 확장 레이어보다 메뉴가 위에 나타나되 화면 밖으로 나가지 않는다.

### 모바일 520px 이하

- 기존 고정 메뉴 위치와 너비를 유지한다.
- 메뉴는 상단 바 아래에 표시되며 가로 여백 12px을 지킨다.

## 시장 화면 정보 구조

### 상단 소개

- 제목: `채용 시장`
- 설명: 공식 채용페이지에서 현재 확인되는 기술 수요를 보여준다고 명시한다.
- 배지: `현재 수집 데이터`를 사용하고 `실시간`, `증가`, `감소` 표현은 사용하지 않는다.

### 경력 필터

- 전체: `career_type` 없음
- 신입: `career_type=new_comer`
- 경력: `career_type=experienced`
- 경력 무관: `career_type=mixed`
- 필터는 링크로 구현해 서버 렌더링과 URL 공유가 가능해야 한다.
- 알 수 없는 값은 전체로 정규화한다.

### 현재 스냅샷

- 공개 공고: `PostingListResponse.total`
- 확인 기술: `SkillStatsResponse.items.length`
- 최근 확인: 불러온 공고 중 가장 최신 `last_verified_at`
- 공고 수와 기술 수를 서로 더하거나 중복 없는 전체 기술 수처럼 과장하지 않는다.

### 기술 수요 순위

- 최대 30개 기술을 공고 수 내림차순으로 표시한다.
- 각 행은 기술명, 카테고리, 공고 수, 필수, 우대, 미분류 건수를 보여준다.
- 막대 길이는 현재 필터 결과의 최댓값 대비 비율이며 추세를 의미하지 않는다.
- 기술명은 `/skill-map?skill=...`로, 관련 공고는 `/jobs?q=...&career_type=...`로 연결한다.

### 최근 확인 공고

- API에서 반환된 실제 공고 중 최대 5개를 표시한다.
- 회사명, 직무, 경력, 지역, 확인일을 사용하고 없는 값은 기존 label formatter로 안전하게 표현한다.
- 각 항목은 `/jobs/:id`로 이동한다.

### 데이터 기준 안내

- 공식 채용페이지의 공개 정보만 사용한다.
- 필수·우대 구분은 공고 문구에서 확인된 분류임을 설명한다.
- 현재 API에는 역사 비교가 없어 변화율을 표시하지 않는다고 명시한다.
- 분석 방법과 데이터 정책 페이지로 연결한다.

## 컴포넌트와 파일 경계

- `app/market/page.tsx`: 검색 파라미터 정규화, 두 API의 병렬 로딩, 리소스 상태 조합
- `features/market/model.ts`: 필터, 최신 확인 시각, 기술 행, 최근 공고를 만드는 순수 함수
- `features/market/market-overview.tsx`: 의미 구조와 링크 렌더링
- `features/market/market-overview.module.css`: 시장 화면 전용 반응형 레이아웃
- `components/app-shell/app-shell.module.css`: 데스크톱 메뉴 세로 배치 규칙
- `e2e/header-brand-lockup.e2e.ts`: 헤더 메뉴와 주요 탐색의 비겹침 회귀 검증 추가

한 파일에 데이터 변환과 대형 JSX를 함께 두지 않는다. 새 상태 관리 또는 차트 라이브러리는 추가하지 않는다.

## 데이터 흐름과 실패 처리

1. 서버 페이지가 필터를 정규화한다.
2. `getPostings({ career_type, limit: 100 })`와 `getSkillStats({ career_type, limit: 30 })`를 병렬 호출한다.
3. 각 요청은 독립적으로 성공 또는 실패 상태가 된다.
4. 순수 모델 함수가 성공한 데이터만 화면 모델로 바꾼다.
5. 공고만 실패하면 기술 수요 순위는 유지하고 공고 요약과 목록에 안전한 오류를 표시한다.
6. 기술 통계만 실패하면 실제 공고 수와 최근 공고는 유지하고 순위 영역에 오류를 표시한다.
7. 둘 다 실패하면 내부 URL이나 상태 코드를 노출하지 않는 전체 오류 안내와 재시도 링크를 표시한다.
8. 빈 응답은 오류와 구분하고 필터 변경 및 전체 공고 탐색 경로를 제공한다.

## 반응형과 시각 원칙

- 데스크톱은 넓은 기술 순위와 좁은 최근 공고 보조 열의 2열 구조를 사용한다.
- 960px 이하에서는 단일 열로 전환한다.
- 모바일 필터는 줄바꿈되며 각 링크의 터치 영역은 최소 44px이다.
- 순위는 표가 아니라 읽기 쉬운 목록으로 구현해 좁은 화면에서 가로 스크롤을 만들지 않는다.
- 홈과 동일한 semantic token, 흰 표면, 약한 테두리, 제한된 보라색을 사용한다.
- 장식용 그래프, 원형 차트, 그라데이션, 강한 그림자를 사용하지 않는다.

## 접근성

- 현재 경력 필터에 `aria-current="page"`를 제공한다.
- 기술 순위는 순서가 의미 있으므로 `ol`을 사용한다.
- 수요 막대는 장식으로 숨기고 수치는 텍스트로 항상 제공한다.
- 오류·빈 상태는 제목과 다음 행동 링크를 포함한다.
- 메뉴 비겹침 수정 후 기존 키보드 닫기와 포커스 복귀 테스트를 유지한다.
- 모든 링크와 버튼은 최소 44px 터치 영역과 `focus-visible`을 제공한다.

## 테스트 전략

### 순수 함수

- 허용·비허용 `career_type` 정규화
- 최신 확인 시각 계산과 잘못된 날짜 무시
- 기술 순위 필수·우대·미분류 기본값
- 경력 필터를 보존한 공고 검색 링크

### 페이지와 컴포넌트

- 두 API가 선택 필터와 지정 limit으로 호출되는지 확인
- 실제 API 값만 렌더링하고 변화율 문구가 없는지 확인
- 한 리소스 실패 시 다른 실제 데이터가 유지되는지 확인
- 빈 상태와 관련 탐색 링크 확인

### 브라우저

- 1536px에서 알림 및 사용자 메뉴 상단이 주요 탐색 하단보다 최소 8px 아래인지 확인
- 390px에서 메뉴가 화면 안에 있고 가로 오버플로가 없는지 확인
- 시장 화면을 1440px, 820px, 390px에서 확인
- 콘솔 오류와 가로 오버플로가 없어야 한다.

## 비범위

- 시계열 저장 및 이전 기간 비교
- 추세 증감률, sparkline, 예측
- 직무·지역·산업 다중 필터
- 로그인 기반 개인화 시장 분석
- 새로운 그래프 엔진 또는 백엔드 API

## 완료 기준

1. 데스크톱 알림·사용자 메뉴와 주요 탐색이 겹치지 않는다.
2. `/market`이 준비 중 문구가 아닌 실제 API 데이터 화면이다.
3. 공고·기술 수치에 mock이나 추정값이 섞이지 않는다.
4. 전체·부분 실패·빈 상태가 서로 구분된다.
5. 기술에서 스킬맵과 관련 공고로 이동할 수 있다.
6. 데스크톱, 태블릿, 모바일에서 가로 넘침 없이 사용할 수 있다.
7. 전체 단위 테스트, 브라우저 테스트, TypeScript, 프로덕션 빌드가 통과한다.
