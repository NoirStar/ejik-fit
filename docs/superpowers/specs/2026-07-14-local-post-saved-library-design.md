# 로컬 커뮤니티 글 저장 보관함 연동 설계

## 문제

사용자가 직접 작성한 브라우저 전용 글도 홈과 상세에서 저장할 수 있지만,
현재 `/career/saved`는 mock 커뮤니티 항목만 복원한다. 따라서 정상적인 `local-*`
저장 ID를 확인 불가 항목으로 잘못 표시하고 정리 동작으로 저장 상태까지 지울 수
있다.

## 목표

- 저장한 브라우저 전용 글을 저장 보관함에서 본문과 함께 다시 연다.
- 내 글과 화면 검증용 mock 글을 같은 사실처럼 섞지 않고 카드마다 출처를 표시한다.
- 같은 탭의 변경과 다른 탭의 `storage` 변경을 즉시 반영한다.
- 삭제된 글이나 현재 mock 목록에 없는 ID만 확인 불가로 분류한다.
- 공고 영역은 계속 현재 공식 공고 API 응답만 사용한다.

## 데이터 경계

- 공고: 기존 `POST /career/saved/data`와 실제 API 재확인 계약을 유지한다.
- 내 글: `ejik-fit:local-community-posts`에서 검증을 통과한 현재 브라우저 데이터만
  읽는다.
- 커뮤니티 예시: 기존 `MOCK_SOCIAL_ITEMS`만 사용하고 예시임을 명시한다.
- 저장 여부: 기존 `ejik-fit:social-interactions.savedPostIds`를 단일 기준으로 쓴다.
- 서버 저장, 계정 동기화, 실제 커뮤니티 API는 만들지 않는다.

## 화면 구조

- 범위 이름을 `커뮤니티 예시`에서 `커뮤니티`로 바꾼다.
- 커뮤니티 섹션 상단에서 내 글은 브라우저 전용이고 예시 글은 mock임을 함께
  설명한다.
- 카드에는 `내 로컬 글` 또는 `예시 콘텐츠` 배지를 표시한다.
- 내 글 카드 작성자는 `나 · 이 브라우저에서 작성`으로 표시한다.
- 저장 해제는 기존 social interaction 저장 계약을 사용한다.
- 확인 불가 안내는 `현재 브라우저 글 또는 예시 목록에서 찾지 못한 저장 글`로
  표현한다.

## 상태와 실패 처리

- 로컬 글 저장소를 읽기 전에는 기존 저장 보관함 로딩 상태를 유지한다.
- 로컬 글 저장소가 손상되거나 접근 불가하면 검증된 글이 없는 것으로 처리하되,
  저장 ID를 임의 콘텐츠로 복원하지 않는다.
- 로컬 글이 삭제되면 전용 구독으로 카드와 개수가 갱신된다.
- 저장 해제가 실패하면 성공 안내를 내지 않고 현재 저장 상태를 유지한다.

## 반응형과 접근성

- 기존 저장 보관함 그리드와 모바일 단일 열을 유지한다.
- 카드 배지는 색뿐 아니라 텍스트로 출처를 구분한다.
- 링크와 저장 해제 버튼은 44px 이상 터치 영역을 유지한다.
- 상태 변경은 기존 `aria-live` 알림에 전달한다.

## 검증 기준

- 모델이 mock, local, 누락 ID를 정확히 분리한다.
- 로컬 글 저장 후 보관함에서 보이고 상세로 이동한다.
- 새로고침과 다른 탭 변경 뒤에도 동기화된다.
- 저장 해제 실패를 성공으로 표시하지 않는다.
- 기존 공고/지원 관리와 mock 커뮤니티 테스트가 회귀하지 않는다.
- 1440px, 820px, 390px에서 가로 넘침과 44px 미만 동작 요소가 없다.
