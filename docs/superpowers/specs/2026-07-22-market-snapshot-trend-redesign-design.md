# 채용시장 스냅샷·추세 재설계

## 상태

- 작성일: 2026-07-22
- 설계 상태: 사용자 승인
- 대상 화면: `/market`
- 선택한 시각화: 순위형 분할 막대

이 문서는 `2026-07-14-interactive-market-dashboard-design.md`의 채용시장 정보 위계, 수요 표, 추세, 개인화 영역을 갱신한다. 기존 기술 분야 필터와 실제 데이터만 사용한다는 원칙은 유지한다.

## 목표

채용시장 화면을 사용자가 현재 기술 수요와 데이터의 최신성, 주간 변화 가능 여부를 짧은 시간 안에 이해하는 시장 관찰 화면으로 만든다.

제품 내 화면 역할을 다음과 같이 분리한다.

- 채용시장: “지금 시장에서 어떤 기술이 명시적으로 요구되고 있으며 실제 주간 수요가 어떻게 변하는가?”
- 스킬맵: “내 보유 기술을 기준으로 다음에 무엇을 준비해야 하는가?”

채용시장에서 개인별 학습 추천과 적합도 분석을 제거한다. 선택한 기술을 개인 관점에서 판단하고 싶을 때만 스킬맵으로 연결한다.

## 사용자에게 제공할 결론

첫 화면은 다음 세 질문에 답해야 한다.

1. 지금 공식 공고에서 명시적으로 많이 요구되는 기술은 무엇인가?
2. 이 수치는 어느 범위와 기준일의 데이터인가?
3. 주간 변화는 판단 가능한 상태인가, 아직 수집 중인가?

사용자가 기술 하나를 선택하면 다음 두 질문에도 답해야 한다.

1. 이 기술은 필수와 우대 중 어느 방식으로 요구되는가?
2. 실제로 어떤 공고에서 확인할 수 있으며, 개인 학습 판단은 어디에서 할 수 있는가?

## 검토한 시각화와 결정

### 선택: 순위형 분할 막대

- 같은 기준선에서 시작하는 막대 길이로 기술 간 수요를 정확히 비교할 수 있다.
- 막대 안에서 필수와 우대 구성을 동시에 읽을 수 있다.
- 기술명을 가로로 유지할 수 있어 데스크톱과 모바일 모두 유리하다.
- 현재 확보된 단일 시점 통계를 그대로 사용하므로 가상 추세를 만들지 않는다.

### 제외: 트리맵 또는 버블맵을 주 시각화로 사용

면적 차이는 시선을 끌지만 비슷한 기술 간 수량 비교가 어렵고 작은 영역의 라벨이 모바일에서 무너진다. 순위형 막대와 같은 질문에 답하므로 중복 시각화가 된다.

### 조건부 사용: 선 그래프

선 그래프는 주간 스냅샷이 최소 4주 이상 축적된 경우에만 보조 영역에 표시한다. 현재 수요 막대를 대체하거나 첫 화면의 주 시각화로 사용하지 않는다.

## 화면 정보 구조

### 1. 제목과 데이터 범위

- 제목: `지금 채용시장의 기술 흐름`
- 설명: 기업 공식 채용 공고에서 확인된 기술 수요임을 명시한다.
- 같은 영역에 분석 공고 수, 기준일, 필터 범위를 표시한다.
- 대한민국 전체 채용시장을 대표한다는 표현은 사용하지 않는다.

### 2. 시장 요약

한 줄 또는 하나의 연결된 패널에서 다음 사실만 보여준다.

- 명시 요구 1위 기술과 공고 수
- 명시 요구 상위 3개 기술명
- 주간 추세 상태: `2/4주 수집 중` 또는 `주간 추세 확인 가능`

기술 카테고리 합계를 별도로 계산하지 않은 상태에서 `AI 중심 시장`, `클라우드 강세`처럼 해석한 문구를 만들지 않는다.

### 3. 필터

- `기술 분야`는 `포함 기술 분야`로 변경한다.
- 도움말: “선택한 분야의 기술이 하나 이상 확인된 공고를 모아, 그 공고에 함께 등장한 모든 기술을 집계합니다.”
- 경력 조건과 포함 기술 분야는 현재 URL 상태를 유지한다.
- 기본 정렬은 `명시 요구 많은 순`이다.
- 보조 정렬은 `전체 등장 많은 순`, `필수 요구 많은 순`, `우대 요구 많은 순`, `기술명 순`을 제공한다.

### 4. 현재 기술 수요

첫 화면의 주 시각화로 상위 8개 기술을 보여준다.

각 행은 다음 순서를 따른다.

1. 순위
2. 기술명과 분야
3. 필수·우대 분할 막대
4. `명시 요구 N건`
5. `전체 등장 N건 · 구분 안 됨 N건`

기본 상태에서 막대 길이는 현재 필터 결과의 최대 명시 요구 수를 기준으로 계산한다.

```text
explicit_count = required_count + preferred_count
bar_width = explicit_count / max_explicit_count
required_segment = required_count / explicit_count
preferred_segment = preferred_count / explicit_count
```

- 막대 길이는 시장점유율이 아니라 현재 1위 기술 대비 길이이다.
- 막대에는 `구분 안 됨`을 포함하지 않는다.
- 명시 요구가 0건이면 빈 트랙과 `명시 요구 0건`을 표시한다.
- 기본 정렬의 동률은 전체 등장 수, 기술명 순으로 해소한다.
- 필수와 우대는 색상뿐 아니라 텍스트 숫자로도 구분한다.
- `상대 수요`라는 별도 막대와 열은 제거한다.

`구분 안 됨`은 다음 의미로 화면에 상시 설명한다.

> 기술은 확인됐지만 원문에서 필수·우대 여부를 확인할 수 없는 공고

기존 `미분류`라는 명칭은 사용하지 않는다.

### 5. 주간 추세

- 데이터는 신규 공고 수가 아니라 각 주 스냅샷 시점의 공개 중인 공고 수이다.
- 제목 또는 설명에 `주간 공개 공고 스냅샷`이라고 명시한다.
- 선의 값은 각 주차의 `required_count + preferred_count`로 계산해 주 시각화와 같은 명시 요구 기준을 사용한다.
- 비교 기술은 최대 3개이며, 현재 선택 기술을 항상 첫 번째로 둔다.
- 나머지 비교 기술은 현재 명시 요구 상위 기술로 채운다.
- 준비된 경우 각 기술의 실제 주차별 명시 요구 공고 수와 `전주 대비 ±N건`을 표시한다. 전주 대비 값은 응답의 마지막 두 주차를 직접 뺀 값이며, 차이가 0이면 `변화 없음`으로 표시한다.
- 예측, 전망, 급상승, 유망 기술 같은 해석은 제공하지 않는다.

4주 미만에서는 다음만 보여준다.

- `현재 N/4주 수집`
- 진행 상태
- 최소 4주가 쌓이면 실제 변화선을 표시한다는 설명

가상의 좌표, 예시 선, 보간한 주차, 상승·하락 문구는 렌더링하지 않는다.

현재 주간 스냅샷에는 경력 조건 차원이 없다. 따라서 추세는 항상 `전체 경력·전체 분야 기준`으로 표시한다. 사용자가 경력 또는 포함 기술 분야 필터를 적용한 경우에도 이 범위를 패널 제목 바로 아래에 반복해, 필터된 수요 막대와 같은 범위로 오해하지 않게 한다. 필터별 과거 추세 저장은 이번 작업에 포함하지 않는다.

### 6. 선택 기술 근거

기술 행을 선택하면 다음 정보를 한 패널에서 갱신한다.

- 전체 등장, 필수, 우대, 구분 안 됨
- 현재 불러온 최대 100개 공식 공고에서 함께 확인된 기술
- 현재 불러온 공고 중 선택 기술이 확인된 공고
- 전체 공고 검색으로 이동하는 링크
- `이 기술을 내 스킬맵에서 보기` 링크

최대 100개 공고에서 계산한 조합과 목록은 전체 시장 집계처럼 표현하지 않는다. 패널에 `현재 불러온 최대 100개 공식 공고 기준`을 표시한다. 전체 수요 수치는 `/api/skills/stats` 결과이고, 공고 예시는 `/api/postings` 응답 범위라는 데이터 경계를 시각적으로 구분한다.

## 상호작용

- 최초 선택 기술은 `명시 요구 많은 순`의 첫 번째 기술이다.
- 정렬을 변경하면 행 순서와 막대 기준값이 함께 바뀐다.
- 필터 전환 중에는 기존 화면을 유지하고 컨트롤에만 진행 상태를 표시한다.
- 필터 결과에 기존 선택 기술이 없으면 새 결과의 첫 기술을 선택한다.
- 기술 선택은 수요 막대, 추세 비교의 첫 기술, 선택 기술 근거를 함께 갱신한다.
- 막대 재정렬 애니메이션은 기존 FLIP 동작을 유지하되 `prefers-reduced-motion`에서는 비활성화한다.
- 초기 8개 이후에는 `전체 N개 기술 보기`로 확장한다.

## 컴포넌트 경계

`MarketOverview`는 필터와 선택 상태를 조정하고 다음 단위에 데이터를 전달한다.

- `MarketPulseSummary`: 범위, 1위 기술, 상위 3개, 추세 준비 상태
- `MarketFilters`: 포함 기술 분야, 경력, 정렬
- `TechnologyDemandChart`: 순위형 분할 막대와 접근 가능한 표 데이터
- `TechnologyTrendPanel`: 최대 3개 기술의 주간 변화 또는 수집 상태
- `SelectedTechnologyEvidence`: 선택 기술 수치, 제한된 공고 근거, 전체 검색 및 스킬맵 링크
- `useMarketTrends`: 선택·비교 기술의 단일 추세 요청과 재시도 상태를 소유하고 요약과 추세 패널에 같은 응답을 제공

기존 `MarketFitInsight`와 `useMarketFit`은 채용시장에서 제거한다. 개인 추천 기능은 이미 스킬맵의 `다음 준비`와 `다음 경로`가 담당하므로 스킬맵 구현은 이번 작업에서 변경하지 않는다.

기존 전역 기술 조합 영역은 독립 섹션으로 유지하지 않는다. 선택 기술과 직접 함께 등장한 기술만 `SelectedTechnologyEvidence` 안에서 보여준다.

## 데이터 흐름

1. 서버 페이지가 현재 URL 필터로 `/api/skills/stats`와 `/api/postings`를 읽는다.
2. 시장 모델이 각 기술의 `explicit_count`와 막대 비율을 계산한다.
3. `MarketOverview`가 정렬된 첫 기술을 선택하고 요약·차트·근거 패널에 전달한다.
4. `useMarketTrends`가 선택 기술을 포함한 최대 3개 기술을 `/market/trend-data`로 한 번 요청한다.
5. `MarketPulseSummary`와 `TechnologyTrendPanel`이 같은 추세 응답을 사용한다.
6. 추세 응답이 준비 상태이면 주간 명시 요구 선과 전주 대비 건수를 표시하고, 수집 상태이면 진행 정보만 표시한다.

새 데이터베이스 테이블이나 외부 차트 라이브러리는 추가하지 않는다. 현재 CSS 막대와 SVG 추세 그래프를 사용한다.

## 상태와 오류 처리

- 기술 통계와 공고가 모두 실패하면 하나의 오류 상태와 재시도 링크를 제공한다.
- 기술 통계만 성공하면 수요 차트와 요약은 유지하고 공고 근거 영역만 오류 상태로 바꾼다.
- 추세 요청 실패는 수요 차트에 영향을 주지 않으며 추세 패널 안에서 다시 시도할 수 있게 한다.
- 필터 결과가 비어 있으면 빈 차트 대신 필터 조건과 초기화 버튼을 보여준다.
- 추세가 4주 미만이면 오류가 아니라 수집 상태로 취급한다.
- 데이터에 없는 값을 0으로 꾸미거나 이전 주차를 복제하지 않는다.

## 반응형과 접근성

### 데스크톱

- 1440×900에서 제목, 데이터 범위, 시장 요약, 필터, 상위 5개 막대가 스크롤 없이 보여야 한다.
- 수요 막대가 가장 넓은 영역을 차지하고, 추세와 선택 기술 근거는 보조 열에 둔다.

### 모바일

- 390px에서 제목, 범위, 요약, 필터, 수요 막대, 추세, 근거 순으로 한 열에 배치한다.
- 가로 스크롤을 만들지 않는다.
- 기술명, 막대, 실제 숫자가 같은 행에서 읽히지 않으면 숫자를 다음 줄로 내려 정확성을 우선한다.
- 모든 컨트롤은 최소 44px 터치 영역을 유지한다.

### 접근성

- 막대 차트에는 기술별 필수·우대·전체 등장 수를 읽을 수 있는 표 또는 동등한 스크린리더 텍스트를 제공한다.
- 선 그래프에는 주차별 실제 수치를 담은 접근 가능한 표를 유지한다.
- 색상 외에 범례, 세그먼트명, 숫자를 함께 제공한다.
- 선택 행은 색상 외에 `aria-current` 또는 동등한 상태 텍스트로 표시한다.
- 로딩과 필터 변경 상태는 `aria-live`로 알리되 반복 발표하지 않는다.

## 객관적 검증 기준

### 5초 정보 위계 점검

1440×900 스크린샷의 첫 화면만 보고 다음 질문에 답할 수 있어야 한다.

1. 데이터 기준일과 분석 범위는 무엇인가?
2. 명시 요구 1위 기술과 건수는 무엇인가?
3. 주간 변화는 확인 가능한가, 수집 중인가?

세 답 중 하나라도 첫 화면에 없거나 서로 다른 숫자로 보이면 실패로 처리한다.

### 자동 테스트

- 모델: 명시 요구 계산, 기본 정렬, 동률 처리, 막대 비율, 0건 처리
- 컴포넌트: `구분 안 됨` 설명, 상위 8개, 더 보기, 선택 변경, 필터 변경, 개인 추천 부재
- 추세: 4주 미만 수집 상태, 4주 이상 실제 선, 전주 대비 건수, 요청 오류
- 범위: 필터 적용 중 추세가 전체 시장 기준임을 명시하는 문구
- 접근성: 범례, 실제 숫자, 선택 상태, 스크린리더용 표
- E2E: 1440×900과 390×844에서 가로 넘침 없음, 첫 화면 정보 위계, 필터 URL 보존, 관련 공고와 스킬맵 링크
- 회귀: 예측·시장점유율·가상 추세·개인 학습 추천 문구가 채용시장에 없음

### 수동 화면 검토

- 막대 길이만 보고 순위를 읽을 수 있는가?
- 색상을 가려도 필수와 우대 건수를 구분할 수 있는가?
- `구분 안 됨`이 낮은 신뢰도의 기술이라는 의미로 오해되지 않는가?
- 빈 추세 영역이 현재 수요보다 더 큰 시각적 비중을 차지하지 않는가?
- 모바일에서 수요 차트가 추세와 근거보다 먼저 노출되는가?

## 제외 범위

- 대한민국 전체 채용시장으로의 통계 보정
- 합격 가능성, 장기 전망, 유망 기술 예측
- 필터별 과거 스냅샷 저장과 추세 집계
- 트리맵, 버블맵, 도넛 차트 추가
- 스킬맵 추천 알고리즘 변경
- 데이터베이스 스키마 변경
