# 이직핏 통합 검색 설계

## 목표

헤더가 약속하는 회사·직무·기술·주제 검색을 실제 탐색 흐름으로 연결한다. 공고·회사·기술은 운영 API에서 확인된 데이터만 사용하고, 커뮤니티 결과는 화면 검증용 예시임을 결과 자체에서 분명히 밝힌다.

## 현재 문제

- 헤더 검색은 통합 검색이라고 표시하지만 `/jobs`로만 이동한다.
- 기업명과 기술을 검색해도 결과의 종류를 구분해 탐색할 수 없다.
- 커뮤니티 태그가 검색 경로로 이어지지 않는다.
- 공고 API의 `total`은 전체 DB 총량이 아니라 현재 제한 범위의 반환 건수이므로 전체 검색량처럼 표현하면 안 된다.

## 정보 구조

경로는 `/search?q={query}&scope={scope}`를 사용한다. 지원 범위는 `all`, `companies`, `jobs`, `skills`, `community`이며 잘못된 값은 `all`로 정규화한다.

검색어가 없을 때는 API를 호출하지 않고 검색 안내와 자주 쓰는 실제 기술 진입 경로를 보여준다. 검색어가 있으면 공고 검색과 기술 통계를 병렬로 읽는다.

전체 결과의 순서는 다음과 같다.

1. 기업: 공고 검색 응답에서 안전한 `company_slug`가 있는 기업을 집계한다.
2. 공고: 공고 검색 API의 현재 반환 항목을 보여준다.
3. 기술: 기술 통계 API의 현재 상위 표본에서 이름이 일치하는 기술을 보여준다.
4. 커뮤니티: 로컬 mock 글에서 제목·본문·태그·카테고리를 검색한다.

각 범위 탭은 URL 링크이며 서버 렌더링을 유지한다. 전체 범위에서는 각 섹션을 요약하고, 개별 범위에서는 현재 응답 안의 결과를 더 많이 표시한다.

## 데이터 진실성

- 공고와 기업 결과는 `getPostings({ q, limit: 100 })`의 성공 응답만 사용한다.
- 기업별 공고 수는 전체 채용 규모가 아니라 현재 검색 응답 안에서 확인된 수라고 표기한다.
- 기술 결과는 `getSkillStats({ limit: 100 })`의 현재 상위 표본만 검색한다. 검색 API가 아닌 통계 표본이라는 한계를 표시한다.
- 커뮤니티 결과에는 항상 `예시 콘텐츠` 배지와 설명을 표시한다.
- 한 API만 실패하면 다른 실제 결과와 mock 커뮤니티는 유지한다. 실패한 섹션은 0건으로 가장하지 않는다.
- 검색어가 없거나 API가 실패했을 때 임의 수치를 만들지 않는다.

## 검색 모델

순수 모델은 검색어를 trim하고 대소문자를 무시해 비교한다.

- 기술: 정확히 일치, 접두 일치, 부분 일치, 수요 건수, 이름 순으로 정렬한다.
- 기업: slug가 있는 응답만 기업명과 slug로 그룹화하고 공고 수, 최근 검증 시각, 대표 기술을 계산한다.
- 커뮤니티: 제목, 요약/본문, 태그, 카테고리, 직무·기업 유형을 검색한다.
- 모든 URL은 기존 계약에서 검증된 id/slug를 `encodeURIComponent`로 인코딩해 만든다.

## 화면 방향

현재 이직핏의 차분한 라이트 디자인과 토큰을 그대로 사용한다.

- 상단: 작은 데이터 출처 레이블, 큰 검색어 제목, 재검색 입력창
- 본문: 데스크톱에서 범위 내비게이션과 결과 열의 2열 구성
- 결과: 카드 남발을 피한 구분선 중심 목록
- 실제 데이터에는 `공식 공고`, `공고 통계` 레이블 사용
- mock 커뮤니티에는 보라색이 아닌 중립적인 예시 배지 사용
- 모바일: 가로 스크롤 범위 탭, 단일 열, 44px 이상 터치 영역, 하단 내비게이션 여백 확보

## 메타데이터와 접근성

- 검색 결과는 질의별 얇은 페이지가 색인되지 않도록 `robots: noindex, follow`를 사용한다.
- 문서 제목은 검색어가 있으면 `“검색어” 검색`, 없으면 `통합 검색`으로 만든다.
- 검색 결과 수와 부분 실패 상태는 제목과 텍스트로 제공하며 색상만으로 구분하지 않는다.
- 범위 링크에는 현재 페이지 `aria-current`를 제공한다.
- 모든 결과 링크와 검색 버튼은 키보드 및 44px 터치 목표를 만족한다.

## 비범위

- 검색 추천 자동완성
- 검색 기록 서버 저장
- 별도 검색 인덱스 도입
- 커뮤니티 서버 검색
- 전체 결과 수를 위한 별도 count API
- 무한 스크롤과 페이지네이션
