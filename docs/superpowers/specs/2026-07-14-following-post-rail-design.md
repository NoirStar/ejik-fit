# 팔로우 중인 예시 글 레일 설계

## 문제

홈에서 예시 작성자를 팔로우하고 `팔로잉` 탭으로 모아볼 수는 있지만, handoff가 요구한 오른쪽 레일의 짧은 재방문 단서가 없다. 사용자가 팔로우 동작의 결과를 즉시 확인하고 다시 해당 글로 갈 수 있는 작은 모듈이 필요하다.

## 데이터 원칙

- 커뮤니티 작성자와 글은 기존 `MOCK_SOCIAL_ITEMS`만 사용한다.
- 헤더와 모듈에 `예시`임을 명시해 실제 사용자나 실시간 활동처럼 보이지 않게 한다.
- 팔로우 선택만 브라우저 로컬 저장소의 기존 `social-interactions` 상태를 사용한다.
- 공고·시장·스킬 수치와 API 호출은 변경하지 않는다.

## 상태와 행동

1. hydration 전에는 팔로우 상태를 단정하지 않고 확인 중 문구를 보여준다.
2. 팔로우한 작성자가 없으면 짧은 빈 상태와 `추천 탭에서 찾기` 버튼을 제공한다.
3. 팔로우한 작성자가 있으면 최신 글 기준 최대 2개를 보여준다.
4. 각 항목은 작성자 이니셜, 이름, 작성 시각, 글 제목과 상세 링크를 가진다.
5. `팔로잉 탭 보기`는 홈의 기존 탭을 선택하고 탭에 포커스를 이동한다.
6. 같은 탭·다른 탭에서 팔로우 상태가 바뀌면 기존 구독 상태를 통해 즉시 갱신한다.

동일 작성자의 글이 여러 개라면 글 단위로 최신순 최대 2개를 표시한다. 로컬 사용자의 글은 팔로우 대상이 아니므로 포함하지 않는다.

## 시각·접근성

- 오른쪽 레일의 기존 약한 카드 계층을 유지한다.
- 항목은 카드 안에 또 카드를 만들지 않고 얇은 구분선으로 나눈다.
- 제목은 2줄 말줄임, 링크와 버튼은 44px 이상 터치 영역을 확보한다.
- 모듈은 제목으로 이름 붙인 `section`, 글 목록은 `list`로 제공한다.
- 모바일에서 오른쪽 레일이 피드 아래로 이동해도 가로 넘침이 없어야 한다.

## 검증

- 컴포넌트 테스트: 확인 중/빈 상태, 팔로우 직후 표시, 최대 2개 최신순, 해제 반영, 탭 이동.
- 브라우저 테스트: 데스크톱 팔로우 → 오른쪽 레일 표시 → 상세 → 팔로잉 탭 → 해제, 모바일 넘침·터치 영역·콘솔 오류.
- 전체 Vitest, TypeScript, audit, build, Playwright 회귀와 운영 배포 스모크.
