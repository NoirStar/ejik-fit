# 이직핏 스킬맵 리디자인

## 목표

기존 스킬 관계 그래프와 API 계약은 보존하면서, 새 커뮤니티 중심 제품 셸 안에서 실제로 탐색할 수 있는 스킬맵으로 재구성한다. 중복 내비게이션과 근거 없는 장식 데이터를 제거하고, 선택한 기술의 공식 공고 근거와 다음 행동이 화면의 중심이 되게 한다.

## 현재 문제

- 글로벌 헤더 아래에 별도 브랜드 레일과 동작하지 않는 도구 메뉴가 한 번 더 노출된다.
- 데스크톱은 네 개의 좁은 열로 분할되어 핵심 그래프와 텍스트가 지나치게 작다.
- 모바일은 그래프, 필터, 분석 패널이 긴 한 줄로 이어지고 고정 하단 내비게이션과 정보 우선순위가 충돌한다.
- `채용 캘린더`가 실제 일정 API가 아닌 생성 패턴을 표시한다.
- 선택 스킬의 `Fit` 값은 전체 보유 기술 비교값이라 선택 기술 지표처럼 오해하기 쉽다.
- 관련 공고가 실제 상세 화면으로 연결되지 않는다.
- `/skills/graph`에서는 글로벌 내비게이션의 스킬맵 활성 상태가 표시되지 않는다.

## 제품 원칙

- 기술, 공고, 관계, 추천 수치는 기존 API 응답에서 받은 값만 표시한다.
- 데이터가 없으면 0이나 임의 수치로 채우지 않고 빈 상태 또는 확인 불가 상태를 보여준다.
- 합격 가능성처럼 해석될 수 있는 점수는 만들지 않는다.
- 선택 기술에는 언급 공고 수, 필수, 우대, 미분류 수를 명시한다.
- 관련 공고는 내부 `/jobs/:id` 상세와 연결하고 공식 원문 근거임을 설명한다.
- 보유 기술은 기존 `ejik-fit` 브라우저 저장 계약을 그대로 사용한다.
- 그래프 상호작용과 API 요청 계약은 보존한다.

## 정보 구조

1. 페이지 제목, 현재 데이터 범위, 분석 방법 링크
2. 기술 검색과 실제 그래프 규모 지표
3. 빠른 기술 선택
4. 관계 그래프
5. 선택 기술 요약과 공식 공고 근거
6. 가까운 기술
7. 다음 준비 기술과 분야 분포
8. 보유 기술 및 그래프 필터

## 반응형 방향

### 데스크톱

- 글로벌 셸 안에 최대 폭을 둔 작업 공간을 사용한다.
- 상단 소개 및 검색 아래에 `필터 / 그래프 / 선택 분석`의 3열 구성을 둔다.
- 그래프 열을 가장 넓게 유지하고 선택 분석은 약 20rem, 필터는 약 17rem로 제한한다.
- 실제 API 기반 다음 준비와 분야 분포는 그래프 아래에 배치한다.

### 태블릿

- 그래프와 선택 분석을 먼저 보여주고 필터는 아래 보조 영역으로 이동한다.
- 그래프 높이는 최소 31rem을 유지한다.

### 모바일

- 제목, 검색, 빠른 선택, 그래프, 선택 분석, 필터 순서로 배치한다.
- 그래프는 화면 너비 전체와 최소 25rem 높이를 사용한다.
- 필터와 보유 기술은 접을 수 있는 기본 HTML disclosure로 제공한다.
- 모든 조작 대상은 최소 44px을 유지하고 하단 내비게이션 높이만큼 여백을 확보한다.

## 상태와 접근성

- API 전체 실패는 페이지 흐름 안의 오류 상태로 표시하고 재시도 링크를 제공한다.
- 필터 결과가 비면 필터 초기화 방법을 안내한다.
- 그래프 canvas에는 동일 데이터를 읽을 수 있는 선택 기술 패널과 공고 목록을 함께 제공한다.
- 현재 기술, 활성 범위, 분야 토글은 `aria-pressed` 또는 명시적인 상태 텍스트를 갖는다.
- 포커스 링, reduced motion, 키보드 제출을 유지한다.
- 빠른 선택과 관련 기술 버튼은 선택 후 그래프와 분석 패널을 함께 갱신한다.

## 제거 대상

- 내부 `ti-rail` 브랜드 및 도구 버튼
- 실제 데이터가 아닌 `CALENDAR_DAYS`와 채용 캘린더
- 선택 기술과 무관한 `Fit` 지표
- 의미 없이 중복된 제품명 영역

## 검증

- 서버 페이지가 seed와 보유 기술을 그대로 API에 전달하는지 테스트한다.
- 합성 캘린더와 중복 메뉴가 렌더되지 않는지 테스트한다.
- 선택 기술의 실제 수치와 관련 공고 링크를 테스트한다.
- 보유 기술 추가·삭제, 범위·분야·검색 필터 상호작용을 테스트한다.
- `/skills/graph`에서 글로벌 스킬맵 내비게이션이 활성화되는지 테스트한다.
- 1440px, 820px, 390px에서 가로 넘침, 주요 타깃, 그래프 높이, 하단 내비게이션 여백을 Playwright로 확인한다.
- 전체 Vitest, Playwright, lint, TypeScript, production build를 실행한다.
