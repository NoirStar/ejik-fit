# 실제 공고 기반 채용 일정 설계

## 목적

채용 일정은 사용자가 공식 공고의 마감일과 최근 기업 채용 활동을 빠르게
확인하는 화면이다. 이직핏이 확인하지 못한 일정을 생성하거나, 마감일이 없는
공고를 임의의 날짜에 배치하지 않는다.

핵심 흐름은 다음과 같다.

```txt
이번 달 마감 분포 → 가까운 마감 공고 → 최근 채용을 시작한 기업 → 공고 상세
```

## 데이터 원칙

- 달력에는 열린 공고 중 `closes_at`이 명시된 공고만 표시한다.
- `closes_at`이 없는 열린 공고는 달력 밖의 `마감일 미표기` 수치로 설명한다.
- 기업 활동은 `first_seen_at`을 기준으로 한 이직핏 최초 확인 건수다.
- 기업이 실제로 공고를 게시한 정확한 시각이라고 표현하지 않는다.
- 날짜 경계는 대한민국 시간(`Asia/Seoul`)으로 해석한다.
- 전체 채용시장 통계나 기업의 전체 채용 규모라고 표현하지 않는다.

## 화면 구조

경로는 `/career/calendar`를 사용한다.

1. 작은 페이지 제목과 데이터 범위 설명
2. 한 줄 요약
   - 선택 월의 마감 공고
   - 앞으로 7일 내 마감
   - 마감일 미표기 열린 공고
   - 최근 14일 신규 공고 기업
3. 월 이동 컨트롤
4. 데스크톱 7열 월 달력
5. 가까운 마감 공고 목록
6. 최근 14일 기업 활동 목록

데스크톱은 달력과 오른쪽 정보 패널을 약 `minmax(0, 1fr) 22rem`으로
배치한다. 980px 이하에서는 오른쪽 패널을 아래로 이동한다.

## 달력 상호작용

- 월 이동은 URL의 `month=YYYY-MM`을 사용해 공유와 새로고침이 가능해야 한다.
- 한 날짜에는 최대 3개 공고를 표시하고 나머지는 `외 N건`으로 요약한다.
- 공고 항목은 상세 페이지로 이동한다.
- 브라우저 또는 로그인 계정에 저장된 공고는 북마크 표시를 한다.
- 관심 기업 공고는 기업 표시를 강조한다.
- 필터는 `전체`, `저장 공고`, `관심 기업` 세 가지이며 전체 페이지를
  다시 요청하지 않고 클라이언트에서 전환한다.

## 최근 기업 활동

최근 기업 활동은 선택 월과 별개로 현재 시각 기준 최근 14일을 사용한다.

각 행은 다음을 보여준다.

- 기업명
- 최근 14일 동안 이직핏이 처음 확인한 열린 공고 수
- 가장 최근 최초 확인 시각
- 가장 가까운 명시 마감일(있는 경우)
- 기업 채용 현황 링크

`채용 확대`, `급성장`처럼 원인이나 규모를 추론하는 표현은 사용하지 않는다.

## 상태 처리

- 로딩: 페이지 뼈대를 유지하는 간단한 skeleton
- 오류: 저장 데이터와 무관하게 서버 데이터를 불러오지 못했다는 메시지와 재시도
- 빈 달: 해당 월에 명시 마감일이 있는 공고가 없다는 메시지
- 활동 없음: 최근 14일에 처음 확인한 열린 공고가 없다는 메시지

## 접근성과 모바일

- 월 이동과 필터는 실제 버튼 또는 링크로 구현한다.
- 현재 날짜와 선택 월 밖 날짜를 텍스트와 `aria-label`로 구분한다.
- 색상만으로 저장 공고와 관심 기업을 구분하지 않는다.
- 모바일에서는 7열 달력을 유지하되 공고 제목 대신 점과 건수를 보여주고,
  날짜별 상세 목록을 달력 아래에 둔다.
- 모바일 고정 내비게이션의 safe-area 여백을 유지한다.

## 비목표

- 사용자의 지원 면접 일정을 서버에 새로 저장하는 기능
- 프로필 기반 공고 추천
- 마감일 추정
- 외부 캘린더 쓰기 권한 또는 Google Calendar 연동
- 기업 채용 의도나 성장률 추론
