# 이직핏 제품 UI 정돈 설계

## 목적

현재 기능과 정보구조를 유지하면서, 검증된 접근성 문제와 반복적인 AI형 시각 문법을 줄여
이직핏을 실제 운영 중인 국내 개발자 이직 서비스처럼 보이게 한다.

## 감사 결과 재판정

### 즉시 반영

- 투명 보라 포커스 링의 대비 실패
- 30–40px에 머무는 모바일 선택·필터 제어
- 모바일에서 두 줄이 되는 공고 탭과 기업 더보기 버튼
- 홈 공식 공고의 굵은 왼쪽 강조선과 카드 안 카드 구조
- 페이지마다 반복되는 장식형 eyebrow
- 같은 상태색을 각 CSS 모듈에서 다시 만드는 문제

### 부분 반영

- 색상 토큰은 이번에 만지는 역할부터 중앙화한다. 모든 CSS의 과거 리터럴을 한 번에
  기계적으로 바꾸지 않는다.
- 도입부 구조는 페이지 목적에 따라 달리하되 공통 제목 크기와 간격은 유지한다.

### 반영하지 않음

- Pretendard 단일 패밀리는 국내 한글 제품 UI에서 결함이 아니다. 새 표시 서체를 억지로
  추가하면 서비스 안정감보다 장식성이 커질 가능성이 높아 유지한다.
- 현재 헤더는 실제 목적지와 모바일 대응을 가진 제품 내비게이션이다. 전면 재구성하지 않고
  포커스·폭·간격만 바로잡는다.

## 접근안 비교

### A. 보수적 제품 정돈 — 채택

기능과 레이아웃 골격을 유지하고, 접근성·위계·반복 장식만 수정한다. 회귀 위험이 가장 낮고
현재 서비스의 장점을 보존한다.

### B. 전면 시각 리브랜딩

표시 서체와 헤더 구조, 페이지별 레이아웃을 크게 바꾼다. 변화는 크지만 사용자가 우려한
“더 이상해지는” 위험과 기능 회귀 범위가 과도하다.

### C. 접근성 수정만 수행

포커스와 터치 규격만 고친다. 가장 안전하지만 홈의 중첩 카드와 반복 eyebrow 같은 시각적
문제는 그대로 남는다.

## 디자인 시스템

프로젝트 루트의 `design.md`를 앱 전체의 기준으로 사용한다. 장르는 modern-minimal이며 기존
보라색을 선택·행동에, `#67A2C5`, `#9BCEC1`, `#FFEBD3`, `#FFB6A6` 계열을 시장 데이터와
조용한 상태 표현에 사용한다. 런타임 색은 OKLCH 토큰으로 관리한다.

Pretendard를 display와 body 역할에 의도적으로 함께 사용한다. 페이지 제목은 28px, 상세
제목은 30px 이내, 섹션 제목은 16–20px 범위를 유지한다.

## 구조

- 홈: Ecosystem Index. 커뮤니티 글, 관련 공식 공고, 시장 신호를 연속 피드와 보조 레일로
  노출한다.
- 시장·공고·커리어: Workbench. 짧은 안내 뒤 필터와 실제 작업 결과를 우선한다.
- 스킬맵: Map / Diagram. 그래프가 주 콘텐츠고 제어·근거가 보조다.
- 정책·방법론·개인정보: Long Document. 카드보다 연속 문서와 여백으로 구분한다.

## 세부 변경

### 공통 접근성

모든 모듈별 포커스 규칙은 불투명한 `--color-focus`를 사용한다. 터치 화면에서 모든 입력,
select, 필터, 아이콘 버튼은 44px 이상이다. focus ring은 애니메이션하지 않는다.

### 홈 공식 공고

피드 내부 공고는 왼쪽 보라 선과 별도 외곽 카드 프레임을 제거한다. `공식 채용 공고` 배지와
한 단계 다른 배경만 유지한다. “내 스택을 추가하면…” 안내는 회색 둥근 상자 대신 얇은 구분선
아래의 보조 문장으로 바꾼다. 공고 상세는 진한 보라색 주요 행동으로 유지한다.

### 공고 탭

시각 라벨은 `전체`, `기술 일치`, `저장`으로 단축하고 숫자를 같은 줄에 둔다. 상세한 “현재
페이지” 범위는 접근성 라벨과 결과 안내가 담당한다. 가로 스크롤이 필요하면 탭 행만 스크롤하며
각 탭 문구는 줄바꿈하지 않는다.

### 시장·데이터 정책 제어

시장 정렬과 추세 비교 select, 데이터 정책 검색과 상태 필터를 모바일에서 44px로 맞춘다.
기업 더보기 버튼의 보이는 문구는 `N개 더 보기` 한 줄이고 남은 전체 수는 접근성 라벨에서
제공한다.

### 페이지 도입부

공고·커리어·기업·스킬맵 상단의 장식형 보라 eyebrow를 제거한다. 데이터 출처와 한계는
설명·요약·정책 링크에 남기므로 신뢰 정보는 사라지지 않는다. 분석 패널의 `분석 기준`처럼
실제 역할을 가진 작은 라벨은 유지한다.

## 상태와 오류

데이터 로딩·오류·빈 상태 로직은 변경하지 않는다. CSS 변경으로 기존 `aria-live`, `role`,
키보드 동작을 손상하지 않는다.

## 검증

- 핵심 동작 테스트만 실행한다: 변경된 컴포넌트의 기존 Vitest, TypeScript 검사, 프로덕션 빌드
- Playwright로 `/`, `/market`, `/jobs`, `/skills/graph`, `/career`, `/companies/nexon`,
  `/data-policy`, `/login`을 확인한다.
- 320, 375, 414, 768, 1440px에서 가로 넘침, 두 줄 제어, 44px 터치 크기, 포커스 표시를 검사한다.
- 홈 공고 카드와 공고 탭은 실제 스크린샷으로 전후 위계를 확인한다.

## 비범위

- API, DB, 인증, 수집기, 라우트 구조, 데이터 매퍼 변경
- 새 폰트·아이콘·애니메이션 라이브러리 설치
- 다크 모드 구현
- 기능 컴포넌트 삭제 또는 교체

## 페이지 인벤토리

- `/`: 커뮤니티·공식 공고·시장 신호 홈
- `/market`: 기술 수요와 공고 근거 시장 작업대
- `/jobs`: 공식 공고 검색·필터·저장·기술 일치 목록
- `/jobs/[id]`: 공고 상세와 공식 원문 근거
- `/skills/graph`: 기술 관계 그래프와 근거
- `/skill-map`: 스킬맵 호환 리디렉션
- `/career`: 보유 기술과 공식 공고 비교
- `/career/account`: 계정과 동기화 관리
- `/career/alerts`: 저장 검색과 새 공고 알림
- `/career/calendar`: 마감일과 채용 활동 달력
- `/career/companies`: 관심 기업 목록
- `/career/questions`: 작성한 커뮤니티 글
- `/career/saved`: 공고·글·지원 단계 저장 보관함
- `/companies/[companyId]`: 기업 정체성과 현재 공개 공고
- `/posts/[id]`: 서버·로컬·기본 커뮤니티 글 상세
- `/search`: 통합 검색 결과
- `/login`: 이메일 인증 계정 진입
- `/data-policy`: 수집 정책과 수집 기업 디렉터리
- `/methodology`: 기술 추출·분류·일치 계산 방법
- `/privacy`: 계정과 로컬·서버 저장 정책
- `/corrections`: 공고 정보 정정 요청 안내

## 성공 기준

- 기존 사용자 흐름과 링크가 유지된다.
- 검증된 대비·터치·줄바꿈 문제가 사라진다.
- 홈 공고가 게시물과 구분되면서도 카드 안 카드처럼 보이지 않는다.
- 페이지 상단에서 장식형 작은 보라 문구가 반복되지 않는다.
- 시장 팔레트가 필터 전환 후에도 동일한 의미를 유지한다.
- 주요 데스크톱·모바일 화면이 이전보다 덜 장식적이고 더 읽기 쉽다.
