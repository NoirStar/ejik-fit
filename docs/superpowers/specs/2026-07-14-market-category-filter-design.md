# 시장 기술 분야 필터 디자인

## 목적

현재 `/market`은 실제 공고·기술 통계를 사용하지만 경력 조건만 제공한다. 기술 추출 시 이미 저장되는 `PostingSkill.category`를 이용해, 사용자가 특정 기술 분야가 확인된 공개 공고와 그 안에서 함께 요구되는 기술을 좁혀 볼 수 있게 한다.

이 필터는 직무 분류나 산업을 새로 추정하지 않는다. 화면에서는 `기술 분야`라고 부르고, 실제 확정 기술 카테고리만 사용한다.

## 데이터 의미

- `category=infra`는 확정 임계값을 통과한 `infra` 기술이 하나 이상 있는 현재 공개 공고를 뜻한다.
- 기술 통계는 그 공고들에 포함된 **모든 확정 기술**을 다시 집계한다. 따라서 인프라 공고에 함께 등장한 Python처럼 다른 카테고리의 기술도 표시한다.
- 공고 목록과 기술 통계는 같은 `category`와 `career_type` 조건을 사용한다.
- 기존 API의 최대 100개 공고, 최대 30개 기술 응답 범위를 유지한다.
- 비교 기간 데이터가 없으므로 변화율을 만들지 않는다.

## URL과 API

- 시장: `/market?category=infra&career_type=experienced`
- 공고: `/jobs?category=infra&career_type=experienced`
- API:
  - `GET /api/postings?category=infra`
  - `GET /api/skills/stats?category=infra`

지원 UI 카테고리는 현재 카탈로그의 13개 값이다: 언어, 프론트엔드, 백엔드, 인프라, 데이터, AI, 보안, 게임, 로보틱스, 모바일, 디자인, 임베디드, QA.

알 수 없는 URL 값은 전체 시장으로 정규화한다. API는 향후 카탈로그 확장을 막지 않도록 길이만 제한하고 문자열을 전달한다.

## 검색 인덱스 경계

기존 Meilisearch 문서는 기술 카테고리를 저장하지 않는다. `q`와 `category`를 함께 요청할 때는 잘못된 결과를 내지 않도록 PostgreSQL 검색으로 내려간다. 기존 카테고리 없는 검색은 현재 인덱스 경로를 유지한다.

## 화면

시장 상단에 두 개의 독립된 필터 그룹을 둔다.

1. 기술 분야: 전체 + 13개 카테고리
2. 경력 조건: 전체, 신입, 경력, 신입·경력

한 그룹을 바꿔도 다른 그룹의 URL 상태를 보존한다. 선택한 분야는 소개 배지와 섹션 설명에 텍스트로 다시 표시한다. 필터는 모바일에서 자연스럽게 줄바꿈하고 모든 링크는 최소 44px 터치 영역을 갖는다.

공고 탐색에는 동일한 기술 분야 select를 추가한다. 시장에서 `관련 공개 공고`로 이동하면 분야와 경력 조건이 그대로 유지되고, 오류 재시도 URL도 모든 필터를 보존한다.

## 접근성 및 상태

- 두 필터 그룹은 서로 다른 `aria-label`을 사용한다.
- 선택 링크는 `aria-current=page`로 표시한다.
- 선택 분야는 색상 외 텍스트와 설명으로도 확인 가능하다.
- 잘못된 URL 값은 빈 결과로 오해하지 않도록 전체로 복구한다.
- 기존 부분 실패, 전체 실패, 빈 상태 계약을 유지한다.

## 제외 범위

- 직무·산업 자동 추론
- 기간별 증감률
- 카테고리별 별도 저장 테이블
- 기존 데이터 재분류
- Meilisearch 전체 재색인
