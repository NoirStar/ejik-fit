# 내 커리어 희망 기술 분야 필터 디자인

## 목적

백엔드의 공고 비교 API는 이미 `domains` 조건을 지원하지만 `/career`에서는 경력 조건만 전달한다. 사용자가 희망 기술 분야를 하나 고르고, 보유 기술과 현재 공개 공고를 그 범위에서 비교할 수 있게 한다.

이 값은 직무나 산업을 새로 추정하지 않는다. 스킬 카탈로그의 도메인 메타데이터를 `희망 기술 분야`라고 명시하며, 합격 가능성이나 학습 순서를 만들지 않는다.

## 실제 데이터 경계

- 분야 선택지는 `GET /api/graph/skills?limit=60`이 반환한 현재 그래프 노드의 `domains`에서만 만든다.
- 각 선택지의 보조 수치는 해당 응답에서 분야에 연결된 기술 노드 수다. 공고 수나 시장 점유율로 표현하지 않는다.
- 보유 기술과 선택 조건은 기존 `POST /api/fit/analyze`에 그대로 전달한다.
- 분야가 선택되면 `domains: [value]`를 보내고, 전체 분야이면 `domains` 필드를 생략한다.
- 그래프 요청이 실패하거나 응답이 잘못되면 전체 분야 비교는 계속 사용할 수 있어야 한다.
- 커뮤니티 데이터나 변화율을 이 기능에 섞지 않는다.

## 화면과 상태

- 기존 `비교 범위` 패널에 `경력 조건`과 `희망 기술 분야`를 함께 둔다.
- 분야 목록은 실제 그래프에서 확인된 분야만 노출하고 `전체 기술 분야`를 기본값으로 둔다.
- 선택지는 사람이 읽을 수 있는 한국어 라벨과 `연결 기술 N개` 근거를 함께 표시한다.
- 선택 변경은 현재 비교 요청에만 적용하며 서버에 사용자 선호로 저장하지 않는다.
- 결과 헤더와 로딩 안내에 선택한 분야와 경력 범위를 텍스트로 다시 표시한다.
- 분야 목록 부분 실패는 짧은 안내로 구분하고 공고 비교 자체를 실패 상태로 만들지 않는다.

## 접근성과 반응형

- 두 select는 각각 고유한 label과 id를 갖는다.
- select는 최소 44px 높이를 유지한다.
- 좁은 화면에서는 두 컨트롤이 한 열로 쌓인다.
- 선택 상태는 색에 의존하지 않고 결과 범위 문구로도 확인할 수 있다.

## 제외 범위

- 분야 다중 선택
- 사용자 계정 또는 서버 선호 저장
- 직무·산업 자동 분류
- 도메인별 공고 수를 새로 추정하는 집계
- 기존 fit 분석 알고리즘 변경
