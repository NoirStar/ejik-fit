# 로컬 커뮤니티 글 설계

## 목표

홈의 커뮤니티 글쓰기에서 만든 글을 새로고침과 화면 이동 뒤에도 다시 읽을 수 있게 한다. 글, 반응, 댓글은 브라우저에만 저장하고 서버 게시나 실제 사용자 커뮤니티로 오해할 표현은 사용하지 않는다.

## 데이터 경계

- 기본 커뮤니티 글과 면접 후기는 계속 명시적인 화면 검증용 mock 데이터다.
- 사용자가 작성한 글은 별도 `ejik-fit:local-community-posts` localStorage 키에만 저장한다.
- 공식 공고, 스킬 수요, 시장 수치는 기존 실제 API 응답만 사용한다.
- 로컬 글은 검색 API나 서버 커뮤니티 데이터에 포함하지 않는다.
- 로컬 글 상세 페이지는 검색엔진 색인을 허용하지 않고 브라우저 저장값이 없으면 정직한 빈 상태를 표시한다.

## 저장 계약

- 글 필드: `id`, `title`, `body`, `tags`, `createdAt`.
- 제목 80자, 본문 1,200자, 태그 최대 4개와 글 최대 30개를 허용한다.
- ID, 날짜, 문자열 길이, 중복 태그와 손상된 JSON을 읽을 때 재검증한다.
- 같은 탭 custom event와 다른 탭 `storage` 이벤트를 모두 구독한다.
- 쓰기가 실패하면 글을 피드에 임시로만 추가하지 않고 작성창과 입력값을 유지하며 실패를 알린다.
- 삭제 성공 시 해당 글의 로컬 공감·저장·댓글도 함께 정리한다.

## 홈과 상세 화면

- 홈은 저장된 로컬 글을 최신순으로 mock/API 피드 앞에 합성한다.
- 로컬 글은 `나`, `이 브라우저에서 작성`, `로컬 글` 경계를 표시한다.
- 제목과 댓글 링크는 `/posts/local-…` 상세로 이동한다.
- 상세 화면은 mock 안내 대신 브라우저 저장 안내를 표시하고, 로컬 댓글과 반응을 지원한다.
- 홈과 상세에서 삭제할 수 있고 삭제 실패를 성공으로 표시하지 않는다.

## 개인정보

- 개인정보 페이지에 저장 키, 저장 범위, 서버 미전송을 설명한다.
- 전체 로컬 데이터 삭제는 로컬 글도 삭제하고 실제 키가 사라졌는지 검증한다.

## 검증

- 정규화, 제한, 손상 복구, 저장 실패, 삭제, 동일·다른 탭 동기화 단위 테스트
- 작성·복원·삭제 실패와 피드 합성 컴포넌트 테스트
- 로컬 상세의 빈 상태, 댓글, 삭제 테스트
- 모바일 작성 → 상세 → 새로고침 → 삭제 E2E
- 전체 Vitest, E2E, lint, build, audit 및 배포 확인
