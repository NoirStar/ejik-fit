# 로컬 작성자 팔로잉 설계

## 목표

홈의 `팔로잉` 탭을 코드에 고정된 상태에서 사용자가 직접 선택하는 상호작용으로 바꾼다. 커뮤니티 글과 작성자는 계속 명시적인 mock 데이터이고, 팔로우 선택만 브라우저에 저장한다.

## 데이터 경계

- 커뮤니티 글, 작성자, 반응 수는 화면 검증용 mock 데이터임을 계속 표시한다.
- 공고와 시장 수치는 기존 실제 API 응답만 사용하며 팔로우 상태로 재가공하거나 임의 생성하지 않는다.
- 팔로우는 로그인 계정이나 서버 관계가 아니라 현재 브라우저의 사용자 선택이다.
- 각 mock 작성자는 표시 이름과 분리된 안정적인 `authorId`를 갖는다.
- 사용자가 브라우저에서 작성한 로컬 글은 자기 글이므로 팔로우 버튼을 표시하지 않고 팔로잉 탭에는 포함한다.

## 저장 계약

- 기존 `ejik-fit:social-interactions` 객체에 `followedAuthorIds: string[]`를 추가한다.
- 기존 저장값에 필드가 없어도 반응·저장·댓글을 잃지 않고 빈 팔로우 목록으로 마이그레이션한다.
- ID는 공백 제거, 중복 제거, 길이 제한, 최대 200개 제한을 적용한다.
- 같은 탭 변경 이벤트와 다른 탭 `storage` 이벤트를 기존 구독 경로로 전파한다.
- 저장소 접근이 실패하면 성공 안내를 하지 않고 반환된 실제 상태를 UI에 반영한다.
- 개인정보의 로컬 데이터 삭제는 기존 social-interactions 키 삭제로 팔로우 상태까지 함께 지운다.

## 홈 UI

- 커뮤니티 카드 작성자 영역에 `팔로우`/`팔로잉` 토글을 둔다.
- 버튼은 44px 이상, `aria-pressed`와 작성자 이름이 포함된 접근성 이름을 갖는다.
- 팔로잉 탭은 `followedAuthorIds`에 포함된 작성자의 mock 글과 자기 로컬 글만 표시한다.
- 아무도 팔로우하지 않았으면 `팔로우한 작성자가 없습니다`라는 전용 빈 상태와 추천 탭 이동 버튼을 제공한다.
- 팔로잉 탭에서 해제하면 해당 작성자의 글은 즉시 사라지고 스크린리더 안내를 남긴다.

## 글 상세 UI

- 글 상세 작성자 영역에서도 같은 팔로우 상태를 읽고 수정한다.
- 상세에서 바꾼 상태는 홈과 다른 탭에 즉시 동기화된다.
- 예시 작성자라는 데이터 안내는 그대로 유지한다.

## 비목표

- 실제 사용자 계정이나 소셜 그래프
- 서버 동기화, 팔로워 수, 알림 생성
- mock 작성자를 실제 인물처럼 표현하는 프로필 페이지
- 공고·시장 추천 알고리즘 변경

## 검증

- 정규화, 구버전 마이그레이션, 토글, 용량 제한, 저장 실패, 동일·다른 탭 동기화 단위 테스트
- 홈 팔로우/해제, 전용 빈 상태, 작성자 단위 필터 컴포넌트 테스트
- 상세에서 팔로우한 뒤 홈 팔로잉 탭에 반영되는 E2E
- 1440px, 820px, 390px에서 버튼 크기, 메뉴 겹침, 가로 넘침, 콘솔 오류 확인
