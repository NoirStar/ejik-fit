# 인터랙티브 채용 시장 대시보드 설계

## 목표

기업 공식 채용 페이지에서 이직핏이 확인한 공고를 바탕으로, 사용자가 현재 기술 수요를 비교하고 자신의 다음 행동과 실제 공고까지 이어갈 수 있는 프로덕션 수준의 채용 시장 화면을 만든다.

참고 이미지는 정보 밀도와 좌우 배치의 기준으로 사용한다. 이미지의 가상 추세, 시장점유율처럼 보이는 퍼센트, 임의 기업 데이터는 복사하지 않는다.

## 검토 결과와 선택한 접근

### 선택: SSR 초기 데이터 + 클라이언트 상호작용의 혼합 구조

- 서버는 현재 필터에 해당하는 기술 통계와 최근 공고를 실제 API에서 읽어 초기 HTML을 만든다.
- 필터는 URL을 유지하되 Next.js transition으로 이동해 전체 화면을 비우지 않는다.
- 정렬, 기술 선택, 최근 공고 전환, 공동 등장 계산은 이미 받은 데이터 안에서 즉시 처리한다.
- 사용자 저장 기술은 기존 `ejik-fit:owned-skills`와 `/skills/graph/fit`을 재사용한다.
- 과거 스냅샷 API가 없으므로 프로덕션에서는 추세 수집 상태만 표시한다.

이 방식은 모든 필터 조합을 미리 내려받는 방식보다 요청량이 적고, 기존의 URL 공유 가능성을 유지하면서 현재의 전체 페이지 이동 느낌을 줄인다.

## 데이터 정확성

- 기본 공고 요청은 최대 100건이므로 반환값이 100건이면 `100건 이상 확인`으로 표시한다.
- 기술 통계 요청 제한을 30에서 100으로 올려 현재 API가 가진 69종 전체를 확인한다.
- 기술별 공고 수는 실제 API 숫자를 그대로 사용한다.
- 필수·우대·미분류 비율은 해당 기술 내부 구성비로만 사용한다.
- 기술 간 막대는 현재 필터의 1위 기술을 기준으로 한 상대 길이이며 시장점유율이 아니다.
- 최근 공고와 기술 조합은 현재 페이지가 받아온 공고 범위에서 계산한다고 명시한다.
- 공동 등장 횟수는 `지원 가능한 공고 증가`로 표현하지 않는다.
- 내 기술 분석의 추천도 API가 반환하는 `supporting_posting_count`를 지원 가능 증가분으로 바꾸어 말하지 않는다.

## 화면 구조

1. 기존 공통 헤더
2. 28~32px의 `채용 시장` 제목과 한 줄 설명
3. 공식 공고 수집 범위를 설명하는 얇은 안내 패널
4. 공고 하한, 기술 수, 마지막 확인일, 출처를 한 패널에 나눈 요약 행
5. 기술 분야·경력 필터와 정렬 컨트롤
6. 좌측 기술 수요 표
7. 우측 추세 수집 상태와 선택 기술 관련 최근 공고
8. 좌측 하단 공동 등장 기술 조합과 내 기술 분석

1440px에서는 `minmax(0, 1fr) 22rem`의 2열 구조를 사용한다. 1200px 미만에서는 오른쪽 패널을 아래로 내리고, 840px 미만에서는 모든 영역을 한 열로 쌓는다.

## 기술 수요 표

- 기본 10개를 표시하고 전체 기술 수를 알려 주는 더보기 링크를 둔다.
- 행은 순위, 기술 아이콘·이름·분야, 공고 수, 요구 조건 누적 막대, 상대 수요 막대, 관련 공고 링크로 구성한다.
- 필수는 브랜드 보라, 우대는 저채도 초록, 미분류는 중성 회색을 사용한다.
- 누적 막대 전체는 각 기술의 필수·우대·미분류 합을 100%로 본 구성비다.
- 상대 수요 막대에는 퍼센트 숫자를 쓰지 않고 `현재 1위 기술 대비`라는 설명을 제공한다.
- 기술 행 선택은 오른쪽 최근 공고와 조합 영역을 갱신한다.

## 아이콘

- `simple-icons`의 개별 아이콘 데이터를 번들에서 선택적으로 가져와 실제 기술 로고를 그린다.
- Python, Docker, Kubernetes, AWS, OpenJDK, Go, PostgreSQL, Redis, Kotlin, Spring, TypeScript, Node.js, React 등 명확한 브랜드만 실제 로고를 쓴다.
- LLM, RAG, Model Serving, SQL 같은 일반 개념은 Phosphor의 중립 아이콘을 사용한다.
- 렌더링 중 외부 URL을 호출하지 않는다.
- 모르는 기술은 분야별 중립 아이콘으로 안전하게 대체한다.

## 상호작용과 애니메이션

- 필터 URL 전환은 `startTransition`과 `router.replace(..., { scroll: false })`로 처리한다.
- 같은 기술의 막대 너비는 CSS transition으로 320ms 동안 변화한다.
- 순위 재배치는 이전·현재 위치를 측정하는 작은 FLIP 훅으로 transform 애니메이션을 적용한다.
- 새 행은 opacity와 translateY로 짧게 진입한다.
- 선택 기술과 최근 공고 목록은 180ms fade를 사용한다.
- `prefers-reduced-motion`에서는 모든 이동·너비 애니메이션을 끈다.
- 필터 중에는 기존 데이터를 유지하고 컨트롤만 진행 상태로 표시한다.

## 추세

- `TechnologyTrendPanel`은 추후 실제 주간 시계열을 받을 수 있는 별도 경계를 가진다.
- 이번 구현의 프로덕션 데이터는 빈 배열이며 `추세 수집 중`, `최근 12주 데이터가 쌓이면 제공`을 표시한다.
- 예시 선 차트나 가짜 상승률은 렌더링하지 않는다.
- 추후 비교 선택은 최대 3개라는 계약을 타입과 UI 문구에 남긴다.

## 내 기술과 기술 조합

- 저장 기술이 없으면 내 커리어에서 기술을 저장하도록 안내한다.
- 저장 기술이 있으면 기존 fit API로 일치 공고 수와 다음 기술 후보를 가져온다.
- 추천 문구는 `이 기술을 요구하는 공고 N건`처럼 API가 실제로 보장하는 의미만 사용한다.
- 조합 추천은 현재 받아온 공고의 기술 배열에서 상위 기술 쌍의 공동 등장 횟수를 계산한다.

## 상태와 접근성

- `/market/loading.tsx`는 제목·요약·표·사이드 패널의 구조를 유지하는 skeleton을 제공한다.
- 부분 API 실패는 살아 있는 영역을 유지한다.
- 완전 실패에는 하나의 alert와 재시도 링크를 제공한다.
- 빈 결과에는 필터 초기화 버튼을 제공한다.
- 기술 행은 실제 button, 공고는 실제 link로 구현한다.
- 범례와 텍스트를 함께 제공하고 색상만으로 상태를 구분하지 않는다.
- 모든 컨트롤은 44px 최소 터치 영역과 `focus-visible`을 가진다.

## 테스트

- 모델 테스트: 100건 하한 라벨, 기술 전체 개수, 정렬, 구성비, 최근 공고 필터, 공동 등장 계산
- 컴포넌트 테스트: 데이터 범위 문구, 추세 수집 상태, 기술 선택에 따른 공고 변경, 내 기술 없음 상태, 오류·빈 상태
- 스타일 계약 테스트: 제목 크기, 2열 폭, 금지된 gradient·과한 그림자 부재, reduced-motion
- E2E: 1440/820/390 레이아웃, 필터 URL 전환, 선택 기술과 공고 갱신, 가짜 추세·시장점유율 부재, 가로 넘침과 콘솔 오류 부재

- 대한민국 전체 시장으로의 통계 보정
