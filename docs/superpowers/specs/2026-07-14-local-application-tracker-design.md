# 로컬 지원 관리 설계

## 목표

저장 보관함의 실제 공고에 사용자가 직접 지원 단계를 기록하고, 지원 중인 공고만 모아 볼 수 있게 한다. 공고 내용은 기존처럼 현재 API에서 다시 확인하고, 사용자가 입력한 단계만 브라우저에 저장한다.

## 데이터 경계

- 공고 제목, 회사, 상태, 기술, 공식 URL은 `/career/saved/data`가 현재 공고 API에서 재확인한 값만 사용한다.
- 지원 단계는 사용자 입력이며 `localStorage`에만 저장한다. 서버 동기화나 실제 지원 완료 여부를 추정하지 않는다.
- 커뮤니티 예시에는 지원 단계를 붙이지 않는다.
- 지원 단계는 저장한 공식 공고에만 연결한다. 저장 해제 또는 확인 불가 공고 정리 시 해당 단계도 함께 삭제한다.
- API 일시 실패 때는 공고 저장과 지원 단계를 유지한다.

## 지원 단계

- 미설정: 지원 관리 대상이 아님
- 지원 준비
- 지원 완료
- 면접 진행
- 처우·오퍼
- 종료

단계는 진행률이나 합격 가능성으로 환산하지 않는다. 사용자가 언제든 선택하거나 미설정으로 되돌릴 수 있다.

## 저장 계약

- 키: `ejik-fit:job-application-stages`
- 값: 공고 ID를 키로, 허용된 단계 문자열을 값으로 갖는 객체
- 공고 ID 길이와 최대 항목 수는 저장 공고 계약과 동일하게 제한한다.
- 알 수 없는 단계, 빈 ID, 너무 긴 ID, 최대 개수 초과 항목은 읽을 때 제거한다.
- 같은 탭 사용자 동작과 다른 탭 `storage` 이벤트를 모두 구독한다.
- 저장소 접근이 차단되거나 JSON이 깨졌으면 빈 상태로 안전하게 복구한다.

## UI

- 저장 보관함 탭에 `지원 관리`를 추가한다.
- `전체`와 `공식 공고`에서는 모든 저장 공고를 보여주고 각 카드에서 지원 단계를 편집할 수 있다.
- `지원 관리`에서는 단계가 설정된 현재 확인 가능 공고만 보여준다.
- 공고 카드의 단계 선택은 44px 이상이며 명시적인 라벨을 갖는다.
- 단계 옆에 `이 브라우저에만 저장`을 표시한다.
- 지원 관리가 비어 있으면 공고를 임의 생성하지 않고 저장 공고에서 단계를 고르도록 안내한다.
- 모바일 탭은 네 항목을 억지로 한 줄에 넣지 않고 2×2로 배치한다.

## 오류와 정리

- 공고 API 전체 오류: 기존 오류 상태를 유지하고 지원 단계는 삭제하지 않는다.
- 일부 404: 사용자가 `확인 불가 항목 정리`를 누를 때 저장 ID와 단계 모두 삭제한다.
- 일부 일시 실패: 단계와 저장 ID를 유지하고 재시도를 제공한다.
- 저장 해제: 저장 ID와 지원 단계를 함께 삭제한다는 것을 카드 버튼 동작과 접근성 안내에 반영한다.

## 비목표

- 실제 기업 지원 시스템 연동
- 지원일, 면접일, 마감 알림 자동 추론
- 서버 계정 동기화
- 합격 확률 또는 단계별 점수
- 가짜 지원 이력이나 샘플 지원 데이터

## 검증

- 저장 정규화, 용량 제한, 손상 복구, 동일·다른 탭 알림 단위 테스트
- 단계 설정/해제, 지원 탭 필터, 저장 해제 연동 컴포넌트 테스트
- 실제 공고 fixture와 명시적 mock 커뮤니티가 섞이지 않는 E2E
- 1440px, 820px, 390px에서 탭·select 44px, 가로 넘침, 콘솔 오류 확인
