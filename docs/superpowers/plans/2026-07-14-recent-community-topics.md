# 브라우저 전용 최근 커뮤니티 주제 구현 계획

1. 최근 주제 저장 계약의 실패 테스트를 먼저 추가한다.
2. 검증·정렬·중복 제거·상한·구독·삭제를 담당하는 로컬 저장 모델을 구현한다.
3. 커뮤니티 상세가 성공적으로 표시된 뒤 기록하는 tracker와 테스트를 추가한다.
4. 홈의 고정 mock 최근 목록을 구독형 최근 기록과 정직한 빈 상태로 교체한다.
5. 로컬 글 삭제와 개인정보 전체 삭제에 최근 기록 정리를 연결한다.
6. 데스크톱 상세→홈→최근 항목→새로고침 E2E와 시각 검토를 실행한다.
7. 전체 단위 테스트, 타입, audit, build, E2E를 통과시킨다.
8. `main`에 푸시하고 GitHub CI, Vercel 웹·API와 운영 최근 기록 흐름을 확인한다.
