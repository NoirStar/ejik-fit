# 브라우저 전용 글 통합 검색 구현 계획

1. 로컬 글 검색·병합 모델의 실패 테스트를 먼저 추가한다.
2. 서버 스냅샷을 변경하지 않는 순수 병합 함수를 구현한다.
3. 검색 화면의 로컬 저장소 hydration·구독과 출처별 카드 테스트를 추가한다.
4. 검색 화면을 클라이언트 경계로 전환하고 문구·배지 스타일을 구현한다.
5. 모바일 작성→검색→새로고침→상세 이동 E2E를 추가한다.
6. 전체 단위 테스트, 타입, audit, build, E2E와 시각 검토를 실행한다.
7. `main` 푸시 후 GitHub CI, Vercel 웹·API, 운영 검색 흐름을 확인한다.
