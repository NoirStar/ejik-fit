# 로컬 작성자 팔로잉 구현 계획

1. 피드 타입에 안정적인 작성자 ID를 추가하고 고정 `isFollowing` 상태를 제거한다.
2. `social-interactions` 저장 계약에 `followedAuthorIds`를 실패 테스트부터 추가한다.
3. 피드 정렬 테스트를 사용자가 팔로우한 작성자 ID 기반으로 교체한다.
4. 홈 카드에 팔로우 토글과 팔로잉 전용 빈 상태를 연결한다.
5. 글 상세 작성자 영역에 같은 로컬 팔로우 토글을 연결한다.
6. 개인정보 설명과 로컬 데이터 삭제 테스트를 팔로우 상태까지 확장한다.
7. E2E에서 상세→홈 팔로잉 동기화와 1440/820/390px 반응형을 확인한다.
8. 전체 Vitest, E2E, lint, build, audit를 실행한다.
9. 코드 리뷰 후 `main`에 병합·푸시하고 운영 배포를 확인한다.
